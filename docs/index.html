SDLC Agentic Harness — Capability Overview

SDLC Agentic Harness is a stage-driven SDLC orchestration engine that forces every code change through a structured, gate-verified pipeline — from intent parsing through architecture, code generation, testing, self-review, and GitHub PR submission — with no stage advancing until its hard completion criteria are satisfied.

Core Capabilities

8-Stage SDLC Pipeline

Sequential pipeline from planning through PR, each stage producing auditable artefacts with hard gate checks.

Planning UI Design Architecture Requirements Coding Testing Review Pull Request

Gate-Based Verification

Every stage enforces programmatic pass/fail criteria — file existence, schema validation, command exit codes, coverage thresholds.

lint build coverage schema validation

Dual Verification

Two verification modalities: code-based (lint, build, test) and AI inference (Gherkin compliance, PRD alignment, PRINCIPLES).

Iterative Coding Loop

The "Agentic Loop" iteratively generates and corrects code until all gates pass, with full context isolation between iterations.

Key Principles

  • Spec-first: Developers specify intent (requiremets) as structured artefacts before any code is written
  • Context isolation: Each stage (and each coding iteration) starts with a fresh LLM context, preventing context rot
  • Stage-gated advancement: No stage can advance until all its hard gate checks pass
  • Prefer code verification: Linting, building, and testing are the primary verification methods
  • AI as secondary: AI inference is used where code-based verification is impractical (currently; Gherkin compliance, PRD alignment)
  • Full audit trail: All artefacts are committed to the repository under sdlc/
  • Portable: The entire pipeline can be copied into any project, and used across coding harnesses;
