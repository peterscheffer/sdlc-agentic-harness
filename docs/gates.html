Gate-Based Verification

Every SDLC stage enforces hard gate checks — programmatic pass/fail criteria that block pipeline advancement. No stage can complete until all its gates pass. Gate results are recorded per-stage in .sdlc_state.json for full auditability.

How Gates Are Run

Gates are executed by the gate_runner.py module, which provides a library of reusable check functions: 

GateCheck("prd_exists", "PRD.md exists",
          lambda: check_file_exists(PRD_PATH))
GateCheck("linter_passed", "Linter command passes",
          lambda: check_command_exits_ok(config.commands.lint))
GateCheck("gherkin_compliance", "Gherkin scenarios implemented",
          lambda: (gherkin_compliance_passed, gherkin_message))

Each check returns a (passed: bool, message: str) tuple. Results are stored in the per-stage GateResults Pydantic model and persisted to the state file.

Gate Failure Behaviour

  • Print the specific failing check with error details
  • Do NOT advance the pipeline
  • Do NOT modify the state file's completed_stages
  • Failed artefacts are removed from disk to prevent stale files
  • The developer sees a clear error and can retry the stage
