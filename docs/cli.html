CLI Command Reference

The harness is invoked via python3 .scripts/sdlc_harness.py from the project root. It supports stage execution, pipeline management, and autopilot mode.

Stage Execution

CommandDescription
--stage planning --feature "<intent>"Start the pipeline for a new feature or change
--stage ui-designRun the optional UI design stage
--stage architectureRun the architecture stage
--stage requirementsRun the requirements stage
--stage codingRun the coding stage (or re-enter the coding loop)
--stage testingRun the testing stage
--stage reviewRun the self-review stage
--stage pr [--force]Submit the GitHub Pull Request

Autopilot Mode

The --autopilot (or -a) flag enables fully automatic pipeline execution. After the requested stage completes successfully, the harness runs all remaining stages without further intervention:

python3 .scripts/sdlc_harness.py --stage planning --feature "add auth" --autopilot

The autopilot loop runs each stage sequentially, checks for failures, updates the PRD after each stage, and generates metrics. If any stage fails (or the review recommends FAIL), the autopilot halts and reports the issue.

Context Injection

The --context flag allows injecting prior conversation context from a file. This context is included in the LLM system prompt and takes precedence over conflicting information in existing artefacts:

python3 .scripts/sdlc_harness.py --stage architecture --context /tmp/discussion.txt

This is the mechanism that preserves developer decisions across terminal sessions — the OpenCode chat exports the conversation to a file, which is then injected into the pipeline stage.

Usage Examples

# Start a new pipeline
python3 .scripts/sdlc_harness.py --stage planning \
  --feature "Add user authentication with JWT tokens"

# Run architecture stage with prior discussion context
python3 .scripts/sdlc_harness.py --stage architecture \
  --context /tmp/arch-discussion.txt

# Run testing and then automatically continue through PR
python3 .scripts/sdlc_harness.py --stage testing --autopilot

# Force PR submission despite FAIL recommendation
python3 .scripts/sdlc_harness.py --stage pr --force

# Check pipeline status
python3 .scripts/sdlc_harness.py status

# Reset pipeline state
python3 .scripts/sdlc_harness.py reset --force
