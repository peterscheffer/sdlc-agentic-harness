Configuration Reference

The harness is configured via sdlc.config.json at the project root. This file must be committed to the repository and is validated using Pydantic v2.

Complete Schema

{
  "$schema": "sdlc-config/v1",
  "default_model": "qwen3.6:35b-a3b",
  "provider": "ollama",

  "stages": {
    "planning":     { "model": "gemma4:latest" },
    "ui-design":    { "model": "qwen3.6:35b-a3b" },
    "architecture": { "model": "gemma4:latest" },
    "requirements": { "model": "qwen3.6:35b-a3b" },
    "coding":       { "model": "qwen3.6:35b-a3b", "max_iterations": 5 },
    "testing":      { "model": "qwen3.6:35b-a3b" },
    "review":       { "model": "gemma4:31b" }
  },

  "commands": {
    "lint": "ruff check .",
    "build": "",
    "test": "python -m pytest tests/ -q",
    "coverage_report": null
  },

  "coverage": {
    "enabled": false,
    "min_percentage": 80
  },

  "github": {
    "base_branch": "main"
  },

  "timeouts": {
    "llm_call_seconds": 900,
    "command_seconds": 120
  }
}

Configuration Sections

SectionFieldRequiredDescription
Top-leveldefault_modelYesFallback model used for any stage without a specific model configured
providerNoLLM provider: ollama, openrouter, or omitted for OpenAI-compatible default
stages<stage>.modelNoPer-stage model override (falls back to default_model)
<stage>.max_iterationsNoMax iterations for the coding loop (default: 5)
commandslintNoShell command for linting (empty = skip)
buildNoShell command for building (empty = skip)
testYesShell command for running tests
coverage_report—NoPath to coverage JSON report for parsing
coverageenabledNoEnable coverage threshold gating (default: false)
min_percentageNoMinimum coverage percentage (default: 80)
githubbase_branchNoTarget branch for PRs, defaults to main
timeoutsllm_call_secondsNoTimeout per LLM call (default: 120)
command_secondsNoTimeout per shell command (default: 300)

Per-Stage Configuration

Each stage can have its own model and provider assignment. Stages omitted from the stages dictionary use default_model. This enables using different models for different cognitive tasks:

StageRecommended UseTypical Model Profile
planningDocument generation, requirement analysisSmaller/faster model
architectureSystem design, decision recordsMid-size model with reasoning
codingCode generation, iterative refinementLargest/most capable model
testingGherkin compliance verification (AI inference)Mid-size capable of code understanding
reviewCode review, quality assessmentLargest model for thorough analysis
