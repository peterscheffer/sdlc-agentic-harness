PRINCIPLES.md Enforcement

The harness supports a developer-authored PRINCIPLES.md at the project root that defines project-specific architectural guardrails. These rules are enforced during the architecture and review stages, providing a mechanism for codifying project conventions, architectural constraints, and quality standards.

PRINCIPLES.md Schema

# PRINCIPLES

## Project
name: MyProject
language: Python
framework: FastAPI

## Architecture Rules

### ARCH-001
description: The pipeline follows a strict sequential SDLC
severity: error
check: All stage transitions are validated against STAGE_ORDER

### ARCH-002
description: Pipeline state is persisted to .sdlc_state.json
severity: error
check: State is read from .sdlc_state.json on every invocation

## Code Quality Rules

### QUAL-001
description: All public functions carry type annotations
severity: warning
check: Public functions have type hints

## Testing Rules

### TEST-001
description: Each stage has hard gate checks
severity: error
check: Gate checks run and must all pass for stage completion

Enforcement Points

PRINCIPLES rules are enforced at two stages in the pipeline:

Architecture Stage

The architecture node loads PRINCIPLES.md, parses all rules, and validates the generated ARCH.md against each rule. Results are written into the ## PRINCIPLES Compliance section of ARCH.md. The stage gate check principles_errors_zero ensures zero severity: error violations before the stage can complete:

[architecture] PRINCIPLES violations detected:
  ✗ ARCH-001: Pipeline follows strict sequential SDLC [FAIL]
  ✓ ARCH-002: State persisted to .sdlc_state.json [PASS]

Review Stage

The review node re-validates the final implementation against all PRINCIPLES rules, including any violations found in the generated code. The review output includes a ## PRINCIPLES Compliance section with file and line references for any violations found.

Graceful Degradation

If PRINCIPLES.md is absent, the harness emits a warning but does NOT block execution — all PRINCIPLES checks are simply skipped. This means the pipeline works out of the box on any project without requiring a PRINCIPLES.md file.

Current Implementation Status

The principles_checker.py module parses PRINCIPLES.md rules and validates the architecture document's declared compliance. The actual code-level check functions (_check_forbidden_pattern, _check_missing_type_hints) are currently stub implementations that return False (no violation detected). The architecture validates that ARCH.md declares compliance in its PRINCIPLES Compliance section, but automated code-level enforcement of principle rules is an area for future enhancement.
