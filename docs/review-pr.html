Structured Self-Review & PR Submission

The final two stages of the pipeline provide a structured self-review followed by automated GitHub PR submission. The review produces a PASS/FAIL recommendation that gates the PR stage, ensuring no unreviewed code reaches the repository.

Review Stage: REVIEW.md

The review stage uses an LLM to analyze the entire change and produce a structured sdlc/review/REVIEW.md with these sections:

SectionContent
## Change SummaryWhat changed, derived from the git diff
## PRD AlignmentLine-by-line check: does the implementation satisfy each task and acceptance criterion in PRD.md?
## PRINCIPLES ComplianceAny violations found in the final code, with file and line references
## Test EvidenceSummary of TEST_REPORT.md: pass/fail counts, coverage percentage
## Recommendationrecommendation: PASS or recommendation: FAIL
## NotesOptional reviewer notes

The recommendation line is parsed programmatically by the gate checker. The line must appear exactly as recommendation: PASS or recommendation: FAIL (colon, space, value).

Review Gates

  • REVIEW.md exists — the file must be written to disk
  • Recommendation present — must contain a recommendation: line with PASS or FAIL value

PR Stage: GitHub Submission

The PR stage automates the entire PR creation process:

  1. Check prerequisites: Verify gh auth status succeeds and current branch is not the base branch
  2. Commit artefacts: Stage and commit all files under sdlc/ with a structured commit message
  3. Build PR body: Construct a structured PR body from pipeline artefacts
  4. Submit via gh CLI: Execute gh pr create with structured title and body
  5. Record URL: Capture and store the returned PR URL in .sdlc_state.json
  6. Mark complete: Set pipeline status to complete

PR Body Structure

## Summary
[First paragraph of sdlc/planning/PRD.md]

## Changes
[Bullet list from REVIEW.md § Change Summary]

## Test Evidence
[Test run summary from sdlc/testing/TEST_REPORT.md]

## Review Outcome
recommendation: PASS

## Pipeline Artefacts
- [PRD](sdlc/planning/PRD.md)
- [Architecture](sdlc/architecture/ARCH.md)
- [Review](sdlc/review/REVIEW.md)
- [Test Report](sdlc/testing/TEST_REPORT.md)

Recommendation Gating

ScenarioBehaviour
Review: PASS → developer runs /prPR is created normally
Review: FAIL → developer runs /prPR is blocked with error message; developer must review REVIEW.md
Review: FAIL → developer runs /pr --forceConfirmation prompt: "Are you sure? (yes/no)". On confirmation, PR is submitted
Review: FAIL → developer runs /codingReturns to the coding loop with review context preserved

PR Gates

  • gh authenticated — gh auth status must exit with code 0
  • Not on base branch — must not be on main or the configured github.base_branch
  • PR created — gh pr create must exit with code 0 and return a URL
